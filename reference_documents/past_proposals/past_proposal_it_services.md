## Understanding of Requirements

Springfield Municipal Authority operates a multi-site network spanning 22 facilities
serving approximately 1,800 employees and providing public-facing digital services.
The current infrastructure, deployed in 2016, faces increasing reliability issues,
lacks the capacity for modern workloads, and does not meet current NIST cybersecurity
framework requirements.

We understand the Authority's priorities to be:
1. **Reliability** — eliminate the recurring outages impacting public services.
2. **Security** — achieve NIST CSF compliance and protect sensitive citizen data.
3. **Performance** — support cloud-based applications and growing bandwidth demands.
4. **Manageability** — reduce the operational burden on the Authority's lean IT team.

## Technical Approach

### Phase 1: Assessment & Planning (Weeks 1–4)
- Conduct comprehensive network discovery and assessment.
- Document current-state topology, traffic patterns, and pain points.
- Deliver Network Assessment Report and detailed Migration Plan.

### Phase 2: Core Infrastructure (Weeks 5–16)
- Replace core routing and switching with Cisco Catalyst 9000 series.
- Implement redundant 10Gbps backbone connecting all sites via existing fiber.
- Deploy SD-WAN overlay for intelligent traffic routing and failover.

### Phase 3: Access Layer & Wireless (Weeks 12–24)
- Upgrade access-layer switches across all 22 facilities.
- Deploy Wi-Fi 6E access points with centralized management.
- Implement network segmentation and 802.1X authentication.

### Phase 4: Security & Monitoring (Weeks 20–30)
- Deploy next-generation firewalls at each site perimeter.
- Implement SIEM with 24/7 monitoring through our SOC.
- Configure Zero Trust network access policies.
- Conduct vulnerability assessment and penetration testing.

### Phase 5: Documentation & Transition (Weeks 28–34)
- Deliver as-built documentation and runbooks.
- Train Authority IT staff (40 hours of hands-on training).
- Transition to steady-state support with defined SLAs.
